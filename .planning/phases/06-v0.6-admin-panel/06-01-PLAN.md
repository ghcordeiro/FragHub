## Objective

Build the complete admin panel for FragHub: dashboard with metrics, player management (CRUD, ban/unban), server lifecycle control, RCON console with safety, plugin configuration editor, and audit logging. Enable admins to manage the platform entirely via web UI without SSH access.

**Purpose:** Deliver operational control to admins while enforcing security boundaries (RCON allowlist, config path traversal prevention, audit trail immutability).

**Output:** 4 backend features + 5 frontend pages + database schema + audit system

## Scope

**Wave 1 (this plan):**

1. **Admin Dashboard & Players API** — Metrics endpoint, player CRUD, ban/unban operations
2. **Server Management API & UI** — Systemd lifecycle, RCON proxy with allowlist, status polling
3. **Plugin Config API & Editor** — File R/W with path traversal prevention, allowlist enforcement
4. **Audit Logging Infrastructure** — Event capture, storage, retention, query UI with filters

**Dependencies:**
- Phase 3: API foundation (auth-api with admin role JWT claims)
- Phase 4: Frontend portal (React 18, Zustand, router)
- Database: MariaDB with existing users, matches, stats tables

## Interfaces

**Admin User Type (from auth-api):**
```typescript
// JWT claims must include:
{
  sub: string;           // user ID
  email: string;
  role: "admin" | "player" | "moderator" | "support";
  iat: number;
  exp: number;
}
```

**Admin API Response Pattern (consistent across features):**
```typescript
{
  data?: any;
  error?: string;
  message?: string;
  pagination?: { page: number; limit: number; total: number };
}
```

**Audit Log Event Structure:**
```typescript
{
  admin_id: number;
  action_type: "ban_player" | "unban_player" | "edit_player" | "create_player" | 
               "change_role" | "rcon_command" | "server_start" | "server_stop" | 
               "server_restart" | "plugin_config_read" | "plugin_config_write";
  target_type?: "player" | "server";
  target_id?: number;
  details: Record<string, any>;  // JSON serialized
  ip_address?: string;
  created_at: Date;
}
```

```typescript
export const RCON_ALLOWLIST = {
  game: [
    /^status$/,                    // game status
    /^mp_roundtime\s+\d+$/,        // game cvars (whitelist pattern)
    /^mp_freezetime\s+\d+$/,
    /^say\s+.+$/,                  // admin commands
    /^ban\s+.+$/,
    /^kick\s+.+$/,
  ],
  blocked: [
    /rcon_password/,               // password change
    /sv_password/,
    /^quit$/,                       // server control (use systemd)
    /^restart$/,
    /;|\&\&|`|\|/,                 // command injection patterns
  ],
};
```

Create `rconService.ts`:
- `sanitizeCommand(cmd)` — removes control chars, newlines, tabs; returns cleaned command
- `validateCommand(cmd, allowlist)` — tests against all blocklist patterns (return false if any match), then tests against allowlist (return true only if matches). Log blocked attempts.
- `executeRcon(server_id, command, timeout=5000)` — loads RCON password from env/config (NEVER expose in response), uses `rcon` npm library, sets timeout, captures output, returns result. Catch timeout → 504, connection error → 503.
- `getServerStatus(server_id)` — calls systemctl is-active, executes RCON status command, parses player count from output, returns {status: "online"|"offline", players_connected, uptime}

- `listServers()` — queries game_servers table (if exists) or returns hardcoded list {cs2, csgo}. For each, calls getServerStatus() in parallel (Promise.all with catch for individual failures). Returns array with live status.

- `executeRconCommand(server_id, command, admin_id)` — sanitizes, validates, executes; logs command + output to audit async; returns {output, command_sanitized}. On validation fail: 400 "Command not allowed". On execution fail: 503/504 depending on error type.

## Threat Model

### Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Admin → API | Admin JWT must carry role=admin claim; API validates before each endpoint |
| Frontend → RCON | Frontend sends only command text; backend holds credentials, validates command, executes safely |
| Admin → Filesystem | Admin can read/write only pre-defined plugin config paths; no path traversal |
| Public → Admin | Non-admin users cannot access any `/admin/*` routes (403 / redirect to login) |

### STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-01 | Escalation of Privilege | Admin role claim in JWT | Mitigate | Verify role claim in requireAdmin() middleware before EVERY handler. Never trust request body for role identification; extract from signed JWT sub claim only. |
| T-06-02 | Injection (RCON) | rconService.executeRcon | Mitigate | Sanitize input (remove control chars, newlines). Test command against blocklist patterns before allowlist. Log all blocked attempts. Use `rcon` library (not shell exec). |
| T-06-03 | Path Traversal | configService.saveConfig | Mitigate | Use `path.resolve()` to get canonical path; verify it starts with allowed directory. Test with payloads: `../../../etc/passwd`, `....//....//etc/passwd`. Log violations. |
| T-06-04 | Information Disclosure | RCON password exposure | Mitigate | Load password from environment/config at startup. NEVER include in response objects, logs, or error messages. Search codebase for 'rcon_password' in responses (test pre-merge). |
| T-06-05 | Denial of Service (RCON) | Rate limiting on RCON endpoint | Mitigate | rconRateLimiter (20 req/min per admin). Return 429 with Retry-After header. |
| T-06-06 | Denial of Service (Audit logs) | Volume of audit logs | Mitigate | Cron job (daily) purges logs older than AUDIT_LOG_RETENTION_DAYS (default 90, min 30). Indices on created_at ensure queries < 200ms. |
| T-06-07 | Insecure Direct Object Reference (IDOR) | Admin edits other admin | Mitigate | Validate that updated player_id exists before modification. Prevent admin from banning self (check admin_id != player_id). Prevent removal of own admin role. |
| T-06-08 | Broken Authentication | Missing JWT validation | Mitigate | requireAdmin() checks token signature, expiry, role claim. Return 401 on missing token, 403 on insufficient role. No endpoints bypass this check. |

All threats with "Mitigate" disposition are addressed in task actions above.
